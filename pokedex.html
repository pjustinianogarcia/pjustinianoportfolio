Pokedex

This code defines a module, pokemonRepository, to manage a Pokémon database using an Immediately Invoked Function Expression (IIFE) for encapsulation. The module maintains an internal pokemonList array and interacts with the PokéAPI to load data about the first 150 Pokémon. The add method validates and adds Pokémon objects to the list, while the getAll method retrieves the entire list. Data is dynamically fetched using the loadList function, which populates pokemonList with Pokémon names and their corresponding detail URLs. The loadDetails function fetches additional information, such as a Pokémon's image, height, weight, and types.

The module includes functionality to dynamically display Pokémon as list items on a webpage. Each list item contains the Pokémon's name, image, and a "Details" button that triggers the display of a modal window. The addListItem method creates and appends these list items to the DOM, loading images asynchronously. When the "Details" button is clicked, the showDetails function fetches detailed information about the Pokémon and displays it in a modal, including attributes like height, weight, and types.

The modal window is created using the showModal function, which dynamically sets its title, body content, and image. The code concludes by loading the Pokémon data via the loadList method and adding them to the UI using the addListItem method for each Pokémon in the repository.
