Pokedex

Overview:

This lightweight web application was developed using HTML, CSS, JavaScript, and Bootstrap. It fetches data from an external API and allows users to view detailed information for each data point. The app simulates a Pokédex by loading data from the PokéAPI and displaying information for the original 150 Pokémon. This is a consumption-only API — only the HTTP GET method is available on resources. This was built as an assignment for the CareerFoundry Full Stack Developer Course.

Process:

The code initializes an IIFE named pokemonRepository to manage a list of Pokémon, storing objects with names and URLs fetched from the Pokémon API (pokeapi.co), limited to the first 150 entries. The loadList function retrieves the list of Pokémon from the API, parses the JSON response, and extracts relevant information such as the name and detailsUrl. The loadDetails function then fetches additional data for each Pokémon, including its image, height, and types. The add function validates input to ensure it's an object with a name property, adding it to the pokemonList array if valid, or logging an error if not. The getAll function returns the entire pokemonList array, allowing access to all Pokémon objects added through the add function. No authentication is required to access this API, and all resources are fully open and available. The code dynamically generates list items and buttons for each Pokémon, styled with Bootstrap, and links them to a modal that displays Pokémon details. The showDetails function loads additional data for the selected Pokémon and triggers the modal's display, while the hideModal function closes it based on certain conditions. The repository's methods—such as add, getAll, addListItem, loadList, and loadDetails—are exposed for external access, enabling the app to load, display, and manage Pokémon details efficiently.
